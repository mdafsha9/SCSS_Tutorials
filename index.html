Syntactically Awesome Style Sheets

Introduction

Sass is a CSS-preprocessor. Sass reduces repetition of CSS and therefore saves time.
Stylesheets are getting larger, more complex and harder to maintain. This is where a CSS-preprocessor can help.
Sass lets we use features that does not exists in CSS like variables, nested rules, mixins, imports, inheritance, built-in-functions and other stuff.

How does Sass works

Sass code does not understand the browsers. Therefore we will need Sass-preprocessor to convert Sass code into CSS code.
This process is called transpiling. So we need to give a transpiller some Sass code and then get some CSS code back.

Variables

Variables as a way to store information that we want to reuse through out stylesheet.
We can store things like color, background-color, font-stacks and any CSS value that we want to reuse.
Sass use $ symbol to make something a variable.

Variables example

$font-stack: Helvetica, sans-serif;
$primary-color: #333;
body{
font:100% $font-stack;
color:$primary-color;
}

Modules

You don't need to write all sass code in a single file. You can split it up however you want @use rule.
This rule loads another sass file as a module, which means you can refers to its variables,
mixins and functions in a sass file namespace based on the filename.

Sass @import and partials

One way to write Dry code and is to keep related in seperate files.We can create a small files
with CSS snippet to include in other sass files. Examples of such files can be reset.scss, variable, fonts, font-size etc.,

Sass @import

Like CSS, sass also supports @import directive.
Sass @import directive allows you to include the content of one file into another.

Modules example

_base.scss
$font-stack:Helvetica, Sans-serif;
color:$primary-color;
body{
font: 100% $font-stack;
color: $primary-color;
}

style.scss
@use 'base';
(or)@import 'base';
.inverse {
background-color: base.$primary-color;
color: white;
}

@mixin directive

The @mixin directive lets we create CSS code that has to be reused through out website.
The @include directive is created, let we use (include) the mixin.

@mixin important-text{
color:red;
font-size:20px;
font-weight:bold;
border:1px solid red;
}
selector{
@include important-text;
}

Mixin example

@extend directive

The @extend directive lets we share a set of CSS properties from one selector to another.
It is useful if you have almost identically styled elements.

.button-basic{
border:none;
padding:15px 10px;
text-align:center;
font-size:16px;
cursor:pointer
}
@extend .button-report{
@extend .button-basic;
background-color:red;
}
@extend .button-submit{
@extend .button-basic;
background-color:green;
color:white;
}
